공개 문서 주소: 포털 / 종합 게임 기획서 / 개발 추적 기획서 / 공개 저장소: hyoseung22/dicespell-docs
운영 규칙: DiceSpell 작업으로 문서가 갱신되면 같은 사이클에서 공개 포털 HTML도 재동기화하고 `scripts/publish_docs_portal.sh`로 재배포해 외부 링크를 최신 상태로 유지한다.
자동화 규칙: 반복 구현 writer는 `OpenClaw DiceSpell execution loop 30m` 하나만 사용하고, 검증을 통과한 변경분만 같은 사이클에서 커밋/푸시한다. 검증 실패, 원격 푸시 실패, 기존 변경 충돌 시에는 강행하지 않고 실행 로그에 원인을 남긴다.
버전 관리 원칙: Codex는 DiceSpell의 기능 구현자이면서 저장소 버전 관리자 역할도 겸한다. 새 요청은 구현 전에 자동화 충돌, 문서 동기화, 검증 재현성, 커밋 단위, 공개 포털 반영, 원격 저장소 안전성에 미치는 영향을 먼저 판단한다.
현재 개발 우선순위: 다음 단계는 `콘텐츠 완성도 + 시스템 유기성 + 재미(FQA)`를 먼저 보고, UX는 그다음, 밸런스는 `최소 플레이 가능 허들과 회귀 방지` 역할로만 운용한다. 즉 지금은 수치 평준화보다 `왜 한 런을 더 하고 싶은가`를 더 중요하게 본다.
작업 선택 기준: 새 작업은 먼저 `이전 선택 -> 현재 보상 -> 다음 기대`를 더 강하게 만드는지, 플레이 감정선과 기억점을 늘리는지, 같은 슬롯에서 필요한 UX 피드백을 최소 보강할 수 있는지 확인한다. 이 기준을 만족하지 못하는 순수 수치 조정은 blocker가 아닌 이상 후순위다.
현재 슬롯 메모: 핵심 문서 세트의 readability refresh를 진행해, 첫 런 온보딩 packet과 `overrunEvent` packet이 포털/노션 첫 화면에서도 `왜 존재하는가`, `누가 어디부터 읽어야 하는가`, `바로 이어서 어떤 문서로 내려가야 하는가`를 즉시 파악할 수 있게 정리한다. 문서 역할과 source-of-truth 내용은 유지하되, 첫 스크린 가독성과 handoff 속도를 높이는 데 집중한다.
검증 규칙: 밸런스 시뮬레이터는 `--grimoires=...` 다중 마도서 입력을 받아 같은 HP/사용군/숙련도 축에서 여러 책을 한 배치로 비교할 수 있어야 하며, 필요할 때는 `--starter-artifacts=` / `--page-count=` override로 시작 유물·페이지 수 가설도 같은 도구 안에서 바로 검증할 수 있어야 한다.

1. 게임 개요

한 줄 요약

야추(Yahtzee) 주사위 족보를 주문으로 변환하여 봉인서 속 몬스터와 싸우는 로그라이크 턴제 전략 RPG.

기본 정보

플랫폼PC (Windows) — Electron
엔진HTML/CSS/JS (순수 웹)
아트 스타일픽셀 아트, 브라운 톤

세계관

중세 판타지 도서관. 플레이어 = 사서이자 주사위 신의 축복을 받은 마도사. 봉인된 마도서의 페이지를 넘기며 봉인 몬스터를 격파, 모든 페이지를 클리어하면 봉인 완료. 클리어 후 "한 권 더?"(OverRun) 가능.

2. 아웃게임

2.1 도서관 (Library)

타이틀 → [게임 시작] → 중세풍 서재 로드. 책장에는 해금된 전체 책 중 3~5권 규모의 랜덤 제안 풀이 배치되고, 별도 보관함(Vault)은 책/유물/몬스터 전체를 훑는 독립 백과사전으로 남는다.

현재빌드 로비의 실제 선택은 4권 제안 풀로 고정돼 있고, `다시 뽑기`로 후보를 새로 갱신할 수 있다. 전체 카탈로그는 선택 흐름과 분리된 `보관함(Vault)`에서 책/유물/몬스터 탭으로만 다시 확인한다.

  • 마우스 오버: 책 제목 + 설명 툴팁
  • 선택 시: 나머지 책 사라짐 → 줌인 → 책 펼침 → 1페이지 시작
  • 보관함: 현재 선택과 무관하게 전체 책/유물/몬스터 정보를 읽는 reference 공간

현재빌드 밸런스 랩: 시작 HP/MP/Gold/리롤 수 등 슬라이더로 즉시 조정 가능한 QA 도구.

3. 인게임 전투

3.1 전투 흐름 (턴 사이클)

1. 턴 시작

  1. MP 회복: BaseManaRegen(2) + 붕괴 티어 보너스 + 유물 보너스
  2. 상태이상 틱 (화상 데미지, 빙결 리롤 감소, 쇄약 실드 감소 등) → 지속시간 -1
  3. 몬스터 Shield -1 감쇠
  4. 리롤 횟수 리셋: Base(3) + 유물 보너스 + 붕괴 보너스
  5. 주사위 잠금 해제, 값 초기화
  6. 몬스터 인텐트 결정 (가중치 + 조건 판정)

2. 주사위 롤링 & 잠금

  • 5개 6면체 주사위 → 잠금 해제된 것만 랜덤 굴림 → 리롤 -1
  • 리롤 0이면 족보 선택 강제
  • 첫 롤 이후 좌클릭으로 개별 잠금/해제 토글

3. 족보 선택 & 주문 발동

  • 최소 1회 롤, MP 충분, 미잠금 슬롯 필요

3.5 상단 합계 보너스 현재빌드

현재 빌드에서는 전투마다 싱글 1~6 점수를 따로 추적하고, 총합이 55점 이상이 되면 비문 보너스 드래프트가 열린다.

  • 대상은 에이스~식스(aces~sixes) 6칸이다.
  • 싱글 족보를 사용할 때마다 해당 칸의 점수를 다시 기록한다.
  • 족보가 일치하면 해당 상단 점수를 기록하고, 불일치로 시전하면 그 칸은 0점으로 덮어쓴다.
  • 같은 칸을 나중에 다시 쓰면 이전 기록은 최신 결과로 교체된다.
  • 총합이 55점 이상이고 아직 이번 전투에서 보너스를 열지 않았다면 즉시 드래프트가 열린다.

단, 주문 해석 직후 패배 판정 또는 전투 클리어가 먼저 발생하면 드래프트는 열리지 않는다. 즉 현재 빌드의 순서는 `주문 해석 -> 처치/패배 확인 -> 전투가 계속될 때만 상단 합계 체크`다.

전투 보상

일반 스테이지 클리어

Gold 획득 + 기본 3택 1, 그리고 후반 normal/오버런 구간에서는 `책 테마 전리품` 1개가 추가된 4택 1이 된다. 이는 late page threshold(책 길이의 약 60% 이후)부터 열려, 현재 책이 던지는 위협 결을 다음 보상 기대와 직접 이어 주는 슬롯이다. 이번 단계에서는 여기에 더해 후반 신규 적을 실제로 잡았을 때 그 적 전리품이 같은 후보군에 직접 끼어들고, 책 클리어 직후에는 이번 세그먼트 동안 모은 전리품 축 일부를 `오버런 전리품 보관함` 3택 1으로 한 번 더 고르게 한다. 또한 정령석/강화석처럼 족보 슬롯을 직접 바꾸는 보상은 적용 직후 어떤 슬롯이 어떻게 변했는지 화면 안에서 즉시 피드백해야 한다.

  • 정령석 (족보 속성 부여)
  • 강화석 (족보 기본DMG 증가)
  • HP 회복: min(maxHP×3~8% + missingHP×15~30%, maxHP×35%)
  • 책 테마 전리품: 현재 책 테마에 따라 late normal/오버런에서만 열리는 추가 선택지. 슬라임 책은 `HP +8, Shield +8, Collapse -8`로 안정/붕괴 정리를, 고블린 책은 `Gold +35 + 무작위 족보 1개 강화`로 경제/화력 환전을, 리치 책은 `MP +2 + 무작위 족보 1개 속성 주입`으로 템포/속성 기대를 제공해 `이번 위협의 결`이 다음 페이지 기대와 바로 이어지게 만든다.
  • 적 전리품: 후반 normal 전투에서 `기록 골렘(420016)` 또는 `붕락 정령(420017)`을 실제로 처치했다면, 보상 후보에 해당 적 전리품 1종이 추가로 섞인다. `균열 석판 조각`은 `Shield +12 + 무작위 족보 1개 강화 + 무작위 족보 1개 대지 주입`으로 방벽 퍼즐 다음 턴을 밀어 주고, `붕락 응축핵`은 `MP +2 + Collapse -12 + 무작위 족보 1개 전격 주입`으로 방금 감당한 붕괴 리스크를 다음 템포 보상으로 바꿔 준다. 오버런에서는 같은 전리품이 강화 버전(`+`)으로 떠 수치가 소폭 올라간다.
  • 오버런 전리품 보관함: 책을 클리어하면 엔딩 화면에서 이번 세그먼트 동안 기록한 특수 적 전리품 + 현재 책의 테마 전리품 중 최대 3개를 다시 보여 주고, 그중 1개를 골라 다음 오버런 진입 직전에 즉시 적용한다. 즉 late normal에서 생긴 기억점을 `책 클리어 -> 다음 권 진입` 경계까지 끌고 가는 선택지다.
  • 현재빌드 UX: 정령석/강화석 적용 직후에는 변경된 족보 슬롯 목록, 새 속성/강화 수치, 연출용 하이라이트를 같은 보상 화면에서 즉시 보여 준다. 현재 빌드는 여기에 더해 변화 전후 값을 카드 안에서 직접 비교해 `없음 -> 전격`, `강화 +0 -> +1` 같은 결과를 바로 읽게 한다.

15. 리스크 & 후속 과제

시스템 리스크

  • 페이지마다 족보 해제 → 장기 긴장감 저하
  • 상점과 보상 역할 일부 중복

콘텐츠 리스크

  • 유물 61종 희귀도별 체감 가치 편차
  • `리치의 일기`의 MP 환급 규칙이 첫 주문 할인/고비용 피해 유물과 겹칠 때 과도한 폭발력을 만들 수 있어 밸런스 로그 검증이 더 필요하다.
  • `금간 해골`의 1회 부활과 빙결/쇄약 동시 압박이 후반 다수전에서 과도한 체감 스파이크를 만들 수 있어 밸런스 로그 검증이 더 필요하다.
  • `잿빛 와이번`의 첫 턴 단일 주문 피해 무효와 무피해 공격 상승이 단일 화력 위주 빌드에 과도한 무력감을 주지 않는지 밸런스 로그 검증이 더 필요하다.
  • 이번 슬롯에 추가한 `붕락 정령`의 `마나 폭발`이 후반 일반전을 흥미롭게 만들지, 혹은 이미 높은 붕괴 구간에서 부당한 마무리 페널티로 느껴지는지 실제 플레이/FQA 기록이 더 필요하다.
  • 이번 슬롯에서 late normal/오버런에 `책 테마 전리품`을 붙여 슬라임=안정, 고블린=경제, 리치=템포·속성 축은 열었지만, 다음 단계에서는 이 보상이 특정 신규 적/이벤트성 페이지와 더 직접 묶였을 때도 같은 기억점이 유지되는지 확인이 필요하다.
  • `태엽 항해일지`의 다음 턴 자원 전환이 `조율사의 나침반`과 겹칠 때 딜/안정성을 동시에 과도하게 밀지 않는지 밸런스 로그 검증이 더 필요하다.
  • `세 겹 장막본`의 턴 시작 MP/리롤 보너스가 `서약의 버클`·`완충 제본끈` 이후 추가 실드 유물까지 겹쳤을 때 지나치게 안정적인 눈덩이를 만들지 밸런스 로그 검증이 더 필요하다.
  • `오탈자 수습본`의 첫 불일치 주문 잠금 무시가 첫 주문 할인/불일치 환급 유물과 합쳐졌을 때 지나치게 안전한 낚시 플레이를 만들지 밸런스 로그 검증이 더 필요하다.
  • `전격 실험지`는 이번 슬롯에서 단일보스 재타격에 `전하 차폐` Shield +4를 더해 `전하 환류 8 + 보스 과전류(+0~6)`와 같은 타이밍에 생존 완충도 돌려주도록 보강했다. HP 100 · 100회 단일 검증에서는 승률 7.4%, 페이지 10 실패율 16.4%, 최종 페이지 클리어율 7.4%를 기록했고, 이어 4권 공통 100회 비교에서도 HP 100 승률 7.8%, 페이지 10 실패율 17.6%로 `토석 압인본` 7.3% / 18.5%를 근소하게 앞질렀다. 다만 여전히 `화염`/`빙결` 선두권보다는 아래이므로 다음 질문은 상위권 추격에 필요한 다음 축이 생존인지 총딜인지다.
  • `토석 압인본`의 추가 파쇄 8과 `남은 적이 없을 때 최대 12까지 깊어지는 붕락`이 `암반 압인석`·대지 다타격 족보와 겹칠 때 후반 실드 적 대응을 지나치게 평면화하지 않는지 밸런스 로그 검증이 더 필요하다. 이번 슬롯 결과만 보면 수치 자체는 과도하게 튀지 않았고, 남은 질문은 `깊은 붕락이 실제로 얼마나 자주 켜지느냐` 쪽으로 이동했다.
  • `balance:sim`에 시작 유물 / 페이지 수 override를 붙여 `종막 사냥 수첩` 병목을 다시 가른 결과, 스타터 교체안보다 `30페이지 길이 자체`가 더 큰 병목으로 확인됐다. 같은 스타터를 둔 채 15페이지 override에서는 승률이 살아났고, 실제 책 길이를 15페이지로 압축한 뒤 단독 24회 배치에서도 HP 75 / 85 / 100 기준 승률이 6.9% / 1.7% / 1.0%까지 회복됐다.
  • 마도서 18권 차별점 설명/튜토리얼 부족
  • 원안 몬스터는 `박쥐`, `해골`, `잿빛 와이번`, `기록 골렘`, `붕락 정령`까지 반영됐고, 다음 후보는 보스형 `골렘` 확장과 오버런/보상 이벤트 연결이다.
