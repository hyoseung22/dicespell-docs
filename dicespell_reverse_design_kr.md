## 1. 제품 개요

### 1.1 한 줄 정의
- `야추 기반 족보 전투`와 `책 페이지 진행형 로그라이크 런`을 결합한 싱글플레이 전략 RPG 프로토타입.

### 1.2 플레이 판타지
- 플레이어는 마도서를 선택하고, 페이지를 넘기며, 주사위를 통해 주문 족보를 완성한다.
- 전투 중에는 MP, 붕괴, 잠금 족보, 상태 이상, 유물 시너지를 동시에 관리한다.
- 전투 종료 후 보상과 상점을 거치며 빌드를 누적 강화하고 최종 보스를 봉인한다.

### 1.3 현재 빌드의 핵심 감정선
- 도서관에서의 시작 책 선택
- 굴림 결과를 읽고 최적 족보를 찾는 전술 판단
- 붕괴와 잠금이 쌓이는 압박감
- 상단 합계 보너스, 속성 주입, 유물 시너지에서 오는 빌드 완성감

## 2. 아웃게임 구조

### 2.1 메인 로비 구조
- 현재 빌드는 별도 타이틀 화면 대신 `도서관`이 메인 로비 역할을 수행한다.
- 도서관은 3개 하위 페이지로 구성된다.
  - 게임 안내
  - 로비 밸런스 세팅
  - 도서관 책 선택

### 2.2 게임 안내 페이지
- 제품 소개 문구
- 핵심 루프 요약
- 런 진행 흐름 요약
- 저장 데이터가 있을 경우 `저장 이어서` 버튼 노출

### 2.3 로비 밸런스 세팅 페이지
- 현재 빌드만의 실험용 기능이다.
- 아래 항목을 런 시작 전에 직접 조정할 수 있다.
  - 기본 시작 HP
  - 기본 최대 MP
  - 기본 시작 MP
  - 기본 시작 Gold
  - 기본 시작 Shield
  - 기본 시작 Collapse
  - 몬스터 HP 비율
  - 상단 합계 보너스 기준값
  - 시작 아티팩트 추가 지급
- 이 값은 로컬 저장되며 재실행 후에도 유지된다.

### 2.4 옵션/설정 구조
- 원안의 전체 옵션 화면 대신 현재 빌드는 상단 바에서 간략 조작만 제공한다.
- 현재 지원 범위
  - BGM On / Off
  - SFX On / Off
  - 해상도 드롭다운
- 해상도 드롭다운 지원 프리셋
  - 현재 창 유지
  - 1366 x 768
  - 1600 x 900
  - 1920 x 1080

### 2.5 저장 구조
- 현재는 `localStorage` 기반 단일 세이브 구조다.
- 원안의 `최대 3개 슬롯` 구조는 아직 구현되지 않았다.
- 저장 데이터가 있으면 도서관에서 이어하기가 가능하다.

## 3. 인게임 전체 흐름

### 3.1 현재 런 기본 루프
1. 도서관에서 마도서 선택
2. 페이지 진입
3. 전투 또는 상점 진행
4. 전투 승리 시 보상 선택
5. 다음 페이지 이동
6. 최종 페이지 보스 처치 시 엔딩

### 3.2 페이지 구조
- 현재 구현은 원안 40페이지 구조가 아니라 `15페이지 또는 30페이지 런`으로 압축되어 있다.
- 마도서마다 총 페이지 수가 다르다.
- 마지막 페이지는 항상 `FinalBoss`가 우선 적용된다.

### 3.3 페이지 배치 규칙

| 구분 | 현재 구현 규칙 |
| --- | --- |
| 일반 전투 | 기본값 |
| 상점 | 3, 8, 13, 18, 23, 28페이지 |
| 보스 | 5의 배수 페이지, 단 마지막 페이지 제외 |
| 최종 보스 | 각 마도서의 마지막 페이지 |

### 3.4 예시
- 15페이지 마도서
  - Shop: 3 / 8 / 13
  - Boss: 5 / 10
  - FinalBoss: 15
- 30페이지 마도서
  - Shop: 3 / 8 / 13 / 18 / 23 / 28
  - Boss: 5 / 10 / 15 / 20 / 25
  - FinalBoss: 30

## 4. 승리/패배 조건

### 4.1 승리 조건
- 현재 선택한 마도서의 마지막 페이지에서 최종 보스를 처치할 경우.

### 4.2 패배 조건
- 플레이어 HP가 0이 될 때
- 붕괴 게이지가 100%에 도달할 때
- 모든 족보가 잠겨 사용 가능한 족보가 없어질 때

## 5. 전투 페이즈

### 5.1 전투 개요
- 전투는 `Roll -> Lock -> Hand -> Spell -> Monster Turn` 구조로 진행된다.
- 각 페이지는 몬스터 구성, 페이지 타입, 보상 종류에 따라 템포가 달라진다.

### 5.2 플레이어 자원
- HP: 생존 자원
- MP: 주문 사용 자원
- Gold: 상점 구매 자원
- Shield: 피해 흡수 자원
- Collapse: 위험 자원
- Reroll: 턴당 주사위 재굴림 자원

### 5.3 턴 시작 규칙
- 기본적으로 턴 시작 시 MP가 회복된다.
- 붕괴 단계와 유물 효과에 따라 회복량과 리롤 수가 변한다.
- 턴 시작 시 플레이어 Shield는 일부 감소한 뒤 유물 효과가 다시 적용될 수 있다.

### 5.4 주사위 굴리기
- 5개의 6면체 주사위를 사용한다.
- 기본 리롤 횟수는 3회다.
- 주사위를 굴린 뒤 필요한 주사위를 잠글 수 있다.
- 리롤 횟수가 0이 되면 사실상 족보 선택 단계로 넘어간다.

### 5.5 주사위 잠금
- 플레이어 턴에 최소 1회 이상 굴린 뒤에만 잠금 가능
- 고정된 주사위는 다음 굴림에서 유지
- 숫자키 `1~5`로도 조작 가능

### 5.6 족보 선택
- 총 13종의 족보를 사용한다.
  - 싱글 1~6
  - 트리플
  - 포
  - 풀하우스
  - 스몰 스트레이트
  - 빅 스트레이트
  - 찬스
  - 야추
- 주사위 결과와 일치하지 않아도 사용 가능하다.
- 단, 불일치 시 피해는 0이며 일부 환급형 유물이 있으면 MP를 돌려받는다.

### 5.7 족보 잠금 구조
- 주문 사용 시 해당 족보 슬롯이 잠긴다.
- 현재 빌드에서는 페이지 전환 시 모든 족보 잠금이 해제된다.
- 원안의 `특정 승리 보상에서 전체 해제`보다 완화된 운영 구조다.

### 5.8 속성 주입
- 각 족보 슬롯에는 무속성 또는 속성이 부여될 수 있다.
- 현재 지원 속성
  - 화염
  - 냉기
  - 전격
  - 대지
- 속성은 보상 페이즈의 정령석 선택을 통해 부여된다.

### 5.9 상단 합계 보너스
- 현재 빌드에서 추가된 확장 시스템이다.
- 싱글 1~6 점수를 누적 합산한다.
- 기준값 초과 시 3개의 랜덤 비문 보너스 중 1개를 선택한다.
- 현재 보너스 풀은 10종이다.
  - HP/MP 회복
  - 광역 피해
  - Gold 획득
  - Shield 획득
  - 붕괴 감소
  - 약화 해제
  - 적 전체 쇄약
  - 잠긴 족보 해제
  - MP 회복
  - 강적 대상 최대 HP 비례 피해

### 5.10 상태 이상
- 화상: 턴 시작 고정 피해
- 빙결: 플레이어면 리롤 감소, 몬스터면 공격력 감소
- 감전: 받는 피해 증가
- 쇄약: 받는 피해 증가, 턴 시작 보호막 감소
- 격노: 몬스터 공격력 증가

### 5.11 속성 상성
- 동일 속성 공격은 효율이 낮다.
- 일부 속성은 특정 속성에 대해 우위 보정을 받는다.
- 현재 구현은 단순 배율 기반 상성 구조다.

### 5.12 몬스터 행동
- 현재 몬스터 인텐트 유형
  - 공격
  - 방어
  - 붕괴 상승
  - 회복
  - 하수인 소환
  - 격노 버프
  - 화상 공격
- 각 몬스터는 턴 시작 시 현재 상황과 가중치에 따라 인텐트를 갱신한다.

### 5.13 전투 승리 처리
- 현재 페이지의 몬스터를 모두 처치하면 보상 페이즈로 전환된다.
- 페이지가 마지막이면 즉시 엔딩으로 넘어간다.

## 6. 상점 페이즈

### 6.1 상점 진입 규칙
- 상점 페이지 도달 시 자동 진입한다.
- 현재는 보드형 연출보다 카드형 전시 구조에 가깝다.

### 6.2 현재 상점 구성
- 유물 4개 랜덤 제시
- Gold로 즉시 구매
- 구매한 유물은 즉시 런에 반영
- 구매 후 `다음 페이지로` 버튼으로 진행

### 6.3 원안 대비 차이
- 원안의 상점은 유물 3개 + 정령석 3개 구조
- 현재 구현은 정령석을 상점이 아니라 `전투 보상`으로 이동시켰다.
- 상점 화면은 말/보드 배치보다 카드 UI 중심으로 재해석되었다.

## 7. 보상 페이즈

### 7.1 일반 전투 보상
- 기본 보상 풀
  - Gold 획득
  - HP/MP 회복
  - 정령석 주입

### 7.2 정령석 보상
- 정령석 획득 시 족보 중 하나를 선택해 속성을 부여한다.
- 부여 속성은 전투와 유물 시너지에 직접 연결된다.

### 7.3 보스 보상
- Boss 또는 FinalBoss 클리어 시 추가 유물 드래프트가 열린다.
- 3개의 유물 중 1개를 선택하여 영구 귀속

### 7.4 페이지 전환 보상
- 현재 구현에서는 다음 페이지 진입 시 족보 잠금이 전부 해제된다.
- 이로 인해 템포는 빨라졌지만 장기 압박감은 원안보다 낮아졌다.

## 10. 유물 구조

### 10.1 유물 수

### 10.2 현재 유물 설계 축

- 첫 주문 할인
- 턴 시작 Shield/MP

- 최대 체력 유지 시 피해

### 10.3 원안 대비 차이
- 원안 데이터시트는 유물 정의가 비어 있거나 축약된 상태였다.
- 현재 프로토타입은 실제 플레이 감각을 만들기 위해 구현 중심으로 유물 종류가 크게 확장된 상태다.

## 11. 붕괴 시스템

### 11.1 역할
- 단순 디버프가 아니라 위험과 보상의 이중 자원으로 작동한다.

### 11.2 현재 붕괴 단계
- 정적
- 요동
- 균열
- 호전
- 격류
- 과부하

### 11.3 현재 효과 방향
- 붕괴가 오를수록 MP 회복과 공격 보너스가 커질 수 있다.
- 동시에 확률성 붕괴 증가와 패배 리스크가 커진다.
- 일부 유물은 저붕괴 유지 또는 고붕괴 하이리스크 운영을 유도한다.

## 12. UI/UX 구조

### 12.1 공통 화면 구조
- 상단: 사운드 토글, 해상도, 세이브, 도서관 이동
- 좌측: 플레이어 상태 또는 보유 유물
- 중앙: 현재 페이즈 메인 무대
- 우측: 페이지 진행, 로그

### 12.2 도서관 화면
- 안내 / 밸런스 세팅 / 책 선택 3분할
- 시작 아티팩트는 카드 선택형
- 해상도는 상단 드롭다운에서 조정

### 12.3 전투 화면
- 좌측 패널 탭
  - 마도사 상태
  - 보유 아티팩트
- 중앙 상단
  - 전투 페이지 정보
  - 턴 수
  - 리롤 수
  - 상단 합계
- 중앙 본문
  - 주사위 테이블
  - 몬스터 시야
  - 기억 회람판
- 우측
  - 페이지 진행
  - 최근 전투 로그

### 12.4 사운드/해상도 UX
- 사운드는 On/Off 단순 토글
- 해상도는 드롭다운형 빠른 접근
- 현재 적용 중인 해상도는 체크 표시와 버튼 내 요약 텍스트로 노출

## 13. 원안 대비 현재 빌드 차이 요약

### 13.1 축소된 부분
- 별도 타이틀 화면, 오프닝, 종료 팝업 미구현
- 세이브 슬롯 3개 구조 미구현
- 전체 옵션 메뉴 미구현
- 보드형 상점 연출 단순화
- 40페이지 장기 캠페인 구조 미구현

### 13.2 확장된 부분
- 로비 밸런스 랩
- 상단 합계 비문 보너스
- 사운드 시스템 전체 적용
- 해상도 프리셋
- 신규 유물 대량 확장
- 신규 마도서 대량 확장
- 상태 이상/속성 시너지 강화

### 13.3 구조적으로 달라진 부분
- 상점의 정령석이 보상 페이즈로 이동
- 족보 잠금 해제가 페이지 전환마다 이루어짐
- 보스 등장 주기가 10페이지 단위가 아니라 5페이지 단위로 재구성
- 최종 페이지 수가 마도서에 따라 15/30으로 유동적

## 14. 현재 빌드의 강점

- 야추 족보와 주문 시스템 결합이 직관적이다.
- 마도서, 유물, 속성, 상단 합계 보너스가 빌드 다양성을 만든다.
- 전투 정보가 카드형 HUD로 재정리되어 데스크톱 플레이에 적합하다.
- 로비 밸런스 세팅을 통해 빠른 수치 실험과 QA가 가능하다.

---

## 15. 현재 빌드의 리스크 및 후속 과제

### 15.1 시스템 측면
- 페이지 전환마다 족보 잠금이 풀려 장기 운영 긴장감이 낮아질 수 있다.
- 상점과 보상 역할이 일부 겹친다.
- 옵션 메뉴가 토글 위주라 접근성 설정 확장성이 부족하다.

### 15.2 콘텐츠 측면
- 유물 수는 많아졌지만 희귀도별 체감 가치 편차 정리는 추가 검토가 필요하다.
- 마도서 수가 늘며 일부 책의 차별점 설명과 튜토리얼 유도가 더 필요하다.
- 몬스터 패턴은 테마성은 살아 있지만 후반 조합 다양성은 더 늘릴 여지가 있다.

### 15.3 UX 측면
- 현재는 정보가 풍부한 대신 첫 플레이어에게는 규칙 학습량이 높다.
- 해상도, 사운드, 저장, 도서관 이동이 상단에 모여 있어 편하지만 장기적으로는 옵션 메뉴 재분리도 고려할 수 있다.

---

## 16. 역기획 결론

- 현재 `DiceSpell Standalone`은 원안의 `야추 기반 전략 전투` 골격을 유지하면서, 데스크톱 프로토타입에 맞게 구조를 압축하고 실험 기능을 크게 확장한 빌드다.
- 특히 도서관-전투-보상-상점 루프, 속성 주입, 붕괴, 유물 누적, 보스 보상은 현재 버전에서도 게임의 정체성을 충분히 설명한다.
- 반면 아웃게임 메타 구조, 옵션 메뉴, 세이브 슬롯, 장기 캠페인 연출은 아직 프로토타입 단계에 머물러 있다.
- 따라서 현재 빌드를 정의하는 가장 적절한 표현은 `원안 핵심 전투 루프를 데스크톱 플레이테스트용으로 재해석한 전략 프로토타입`이다.
