# DiceSpell 게임 플로우 리디자인 패킷

## 1. 목적

이 문서는 현재 DiceSpell의 `도서관 -> 전투 -> 보상 -> 상점 -> 엔딩/오버런` 구조를 유지하면서도, 플레이 감각을 더 `게임 보드형`, `런 기반`, `한 번 더 하고 싶게 만드는 흐름`으로 재해석하기 위한 시각/구조 설계 패킷이다.

이번 리디자인은 기존 GDD의 시스템을 뒤엎는 것이 아니라 다음을 강화하는 데 목적이 있다.

- 선택 직후 기대감이 올라가는가
- 현재 페이지의 위협이 다음 페이지 보상 기대와 이어지는가
- 플레이어가 지금 `런 안에서 어디에 서 있는지` 즉시 읽히는가
- 책 완주와 오버런이 `메뉴 종료`가 아니라 `다음 승부처`처럼 느껴지는가

## 2. 레퍼런스 번역 원칙

### Balatro에서 가져올 것

- 한 판의 정체성을 한눈에 보여 주는 `테이블/보드` 감각
- 현재 런의 핵심 자원과 기대 보상이 크게 보이는 무대형 레이아웃
- 작은 수치보다 `지금 무엇이 특별한가`가 먼저 읽히는 강조 방식
- 보상 선택이 즉시 다음 플레이 상상을 만들게 하는 카드형 제시

### Slay the Spire 2에서 가져올 것

- 런 진행 위치가 또렷한 `경로/노드` 구조
- 다음 전투, 휴식, 상점, 이벤트의 기대가 미리 읽히는 진척 표현
- 전투와 보상, 보스 구간이 한 런 안에서 연결된 여정처럼 보이는 흐름
- 한 화면 안에서도 우선순위가 명확한 레이아웃 계층

### DiceSpell이 유지해야 할 것

- 양피지, 서고, 청동, 어두운 잉크, 봉인서라는 세계관
- 야추 족보와 주문 전투의 직접 결합
- 붕괴(Collapse)와 페이지 진행의 긴장감
- 책을 선택하고, 봉인하고, 한 권 더 이어 가는 구조

## 3. 현재 플로우의 문제 정의

현재 홈/도서관 표면은 기능적으로는 충분하지만, 다음 문제가 있다.

- `프로토타입 아틀리에`처럼 읽히며 실제 런의 시작점보다 개발 허브처럼 보인다.
- 플레이어가 `이번 런의 테마`, `이 책이 어떤 플레이 감각을 주는가`, `첫 페이지에 들어갔을 때 무엇을 기대해야 하는가`를 강하게 느끼기 어렵다.
- 페이지 구조와 보상 기대가 분절되어 보여, 한 런의 여정이라는 감각이 약하다.
- 엔딩/오버런이 `다음 권으로 이어지는 게임적 문턱`보다 `결과 정산 화면`처럼 읽힐 위험이 있다.

## 4. 목표 플로우

리디자인 후 DiceSpell의 대표 흐름은 아래처럼 읽혀야 한다.

1. `도서관 / 책 선택`
   - 책은 메뉴 항목이 아니라 `이번 런의 판`처럼 느껴진다.
   - 책마다 위협, 보상 성향, 초반 운용 힌트가 한 번에 읽힌다.

2. `페이지 아틀라스 / 경로 보드`
   - 플레이어는 현재 페이지와 다음 2~3개의 갈림, 보스 구간, 상점/이벤트를 한눈에 본다.
   - 페이지가 단순 번호가 아니라 `작은 여정`처럼 느껴진다.

3. `전투 테이블`
   - 중앙은 현재 승부, 상단은 이번 페이지 stakes, 측면은 다음 위협과 현재 런 상태를 보여 준다.
   - 주사위/족보/붕괴는 계산 UI가 아니라 `긴장 장치`처럼 보여야 한다.

4. `보상 / 상점 / 전리품`
   - 보상은 단순 3택이 아니라 `이번 위협에서 무엇을 건져 다음 페이지를 어떻게 바꿀 것인가`로 읽힌다.
   - 책 테마 전리품, 특수 적 전리품, 상점 특가가 모두 같은 축에서 비교된다.

5. `보스 문턱 / 오버런`
   - 책 완주는 종료가 아니라 `다음 권을 여는 의식`처럼 느껴진다.
   - 이번 세그먼트에서 모은 기억점이 다음 권 진입에 직접 연결된다.

## 5. 화면 구조 제안

### A. Library / Choose Your Tome

기존 GDD 기준: `2.1 도서관`, `2.2 마도서`

화면 역할:

- 이번 런의 정체성을 결정하는 입구
- 책 카드 3장을 단순 리스트가 아니라 `운명 선택`처럼 보여 줌

핵심 요소:

- 중앙 책 3장: 테마, 위험, 완주 보너스, 시작 유물, 첫 보상 기대
- 우측 `Run Forecast`: 예상 길이, 첫 상점, 첫 보스, late 전리품 성향
- 하단 `왜 이 책을 고르나`: 초반 운영 요약

### B. Page Atlas / Route Board

기존 GDD 기준: `8. 상점 & 보상`, `9. 스테이지 구조`

화면 역할:

- 현재 런의 `진행 위치`를 강하게 보여 줌
- 다음 페이지 기대를 시각적으로 연결

핵심 요소:

- 페이지 노드 보드
- 현재 페이지 강조
- 다음 2~3개 노드 미리보기
- 보스 문턱, 상점, 특수 적 출현 가능성 표시
- 우측 `Tension Track`: 붕괴, 골드, 속성 주입, 상단 합계 진행

### C. Battle Table / Spell Arena

기존 GDD 기준: `3. 인게임 전투`, `5. 붕괴 시스템`, `6. 속성 시스템`

화면 역할:

- 계산 화면이 아니라 `현재 한 수`에 집중하는 무대

핵심 요소:

- 중앙 전장: 적 intents, 현재 페이지 stakes, 보드성 연출
- 하단 주사위 트레이와 족보 패널
- 좌측 플레이어/유물/상태
- 우측 몬스터 trio와 다음 위험
- 상단 `Page Stakes`: 이번 페이지 보상 힌트, 책 테마 압력

### D. Reward Market / Curate the Run

기존 GDD 기준: `7. 유물`, `8. 상점 & 보상`

화면 역할:

- 선택 즉시 다음 전투 이미지가 떠오르게 만드는 보상 연출

핵심 요소:

- 기본 보상 3장
- 책 테마 전리품 1장
- 특수 적 전리품 슬롯
- 상점 특가 또는 다음 페이지 준비 힌트
- 하단 비교 요약: 지금 고르면 무엇이 바뀌는가

### E. Boss Threshold / Overrun Oath

기존 GDD 기준: `2. 아웃게임`, `8. 상점 & 보상`, `9. 스테이지 구조`

화면 역할:

- 책 완주를 `의식`, 오버런을 `다음 챕터 제안`처럼 보여 줌

핵심 요소:

- 봉인한 책의 결과
- 오버런 전리품 보관함 3택 1
- 다음 권 진입 CTA
- 이번 런 기록 / 새 해금 / 다음 리스크

## 6. 시각 원칙

- 배경은 평면이 아니라 `서고 + 카드 테이블 + 봉인진`이 겹친 분위기
- 정보는 패널이 많더라도 `주 무대 1곳 + 보조 정보 2곳` 구조를 유지
- 숫자는 작게, 의미는 크게
- 노드, 카드, 책, 전리품은 모두 `손에 잡히는 물건`처럼 표현
- 붕괴, 보스 문턱, 오버런은 별도 색/광원으로 긴장점 구분
