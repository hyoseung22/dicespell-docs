Portal Structure

1. 기준 기획서

전체 구조와 핵심 시스템의 기준점을 제공하는 상위 문서입니다.

2. 실행 추적 문서

현재 스냅샷, 우선순위, 리스크, 변경 이력을 따라가는 운영 문서입니다.

3. 전달 패킷

프로그래머, UI, 아트 담당자가 실제 작업에 들어갈 수 있게 분리된 handoff 사양입니다.

Documentation Intent

핵심 시스템 전체 구현이 최종 목표

현재 자동화의 직접 목적은 기능을 무작정 추가하는 것이 아니라, 핵심 시스템 전체 구현이 가능하도록 문서를 handoff-ready 상태까지 밀어 올리는 것입니다.

Handoff Standard

역할별 전달 가능성 기준

각 패킷은 가능한 한 프로그래머용 구현 기준, UI 상태 정의, 아트 에셋 요구사항, 검수 기준을 분리해 실제 제작자가 바로 착수할 수 있는 수준을 목표로 합니다.

Foundation

기준 문서

프로젝트의 기준 방향과 전체 시스템 정의를 잡아주는 상위 허브입니다. 포털의 다른 문서들은 모두 이 기준에서 파생됩니다.

Master / Source

Primary GDD

종합 게임 기획서

DiceSpell의 원안과 현재 빌드 차이를 한 문서에서 읽을 수 있도록 묶은 기준 기획서입니다. 전투 규칙, 몬스터, 아티팩트, 메타 구조, 책 구조 등 전체 게임의 설계 축을 정리합니다.

  • 전체 게임 루프와 핵심 시스템 기준점
  • 원안 대비 현재 빌드 반영 상태
  • 추후 handoff 문서들이 참조하는 상위 기준 문서
